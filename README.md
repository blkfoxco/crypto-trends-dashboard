# Crypto Google Trends + BTC Price Dashboard

**Live dashboard →** https://crypto-trends-dashboard.vercel.app

An interactive chart overlaying crypto-related Google Trends search interest against Bitcoin price over the last 5 years. Updated weekly every Monday at 01:00 GMT+7.

## What It Shows

- **Google Trends** — normalised search interest for crypto-related terms (Bitcoin, Ethereum, crypto, etc.) over the past 5 years, anchored to Bitcoin for cross-batch comparability
- **BTC Price** — BTCUSDT closing price from Binance, plotted on a linear USD scale

The chart helps visualise the relationship between public search interest and Bitcoin price cycles — useful as a directional signal for market sentiment.

## How It Works

- Data is fetched via [pytrends](https://github.com/GeneralMills/pytrends) (Google Trends) and the Binance public API
- Trends are normalised using Bitcoin as an anchor term to enable consistent comparison across batches
- The dashboard is a single static HTML file generated by a Python script and deployed to Vercel
- A cron job regenerates and redeploys the file every Monday morning

## Stack

- **Python** — data fetching and chart generation (`trends.py`)
- **Plotly** — interactive charting (embedded in HTML)
- **Vercel** — static hosting and deployment

## Files

| File | Description |
|---|---|
| `index.html` | Generated dashboard (do not edit manually) |
| `trends.py` | Main data fetch + chart generation script |
| `trends_html.py` | HTML rendering helpers |
| `vercel.json` | Vercel deployment config |

## Running Locally

```bash
pip install pytrends requests plotly
python trends.py
# Opens index.html in your browser
```

## Caveats

Google Trends represents *relative* search popularity, not absolute volume. The anchor normalisation approximates cross-batch comparability but may introduce minor distortions. Use as a directional signal, not a precise metric.
